## Tổng quan quá trình thực tập

Trong suốt thời gian thực tập tại **Amazon Web Services Vietnam** từ **12/08/2025** đến **12/11/2025** với vai trò **FCJ Cloud Intern**, tôi đã có cơ hội học hỏi, rèn luyện và áp dụng kiến thức đã được trang bị tại trường vào môi trường làm việc thực tế trong lĩnh vực Cloud Computing.

Tôi đã tham gia **xây dựng hệ thống Serverless Student Management Platform trên AWS**, bao gồm thiết kế kiến trúc serverless với 14 dịch vụ AWS (Lambda, DynamoDB, API Gateway, Cognito, S3, CloudWatch, EventBridge, SES, Amplify, CodePipeline, CodeBuild, CodeDeploy, Route53, WAF), phát triển 17 REST API endpoints, triển khai hệ thống chat với polling mechanism, và thiết lập CI/CD pipeline với GitLab và AWS CodePipeline.

Qua đó, tôi đã cải thiện đáng kể các kỹ năng:
- **Kỹ năng kỹ thuật:** AWS serverless architecture, Infrastructure as Code (AWS CDK), NoSQL database design (DynamoDB), GraphQL API development, real-time applications, event-driven architecture, CI/CD automation
- **Kỹ năng mềm:** làm việc nhóm với parallel development (Backend/Frontend teams), quản lý dự án Agile, documentation kỹ thuật, problem-solving trong môi trường production
- **Kỹ năng chuyên môn:** AWS Well-Architected Framework, security best practices, cost optimization, performance tuning, testing strategies

Về tác phong, tôi luôn cố gắng hoàn thành tốt nhiệm vụ đúng timeline (5 tuần với 5 giai đoạn rõ ràng), tuân thủ coding standards và security guidelines của AWS, và tích cực trao đổi với mentor cũng như team members để nâng cao hiệu quả công việc thông qua daily standups và weekly sprint reviews.

## Điểm mạnh đã phát huy

### 1. Kiến thức và kỹ năng chuyên môn
- Nắm vững kiến thức về AWS serverless architecture và các dịch vụ core (Lambda, DynamoDB, API Gateway, Cognito)
- Thành thạo việc thiết kế và triển khai RESTful APIs với performance cao (<500ms response time)
- Có khả năng áp dụng Infrastructure as Code (AWS CDK) để quản lý hạ tầng một cách hiệu quả
- Hiểu rõ về security best practices: IAM least privilege, encryption at rest/transit, Cognito authentication

### 2. Chủ động và sáng tạo
- Chủ động setup parallel development workflow giữa Backend và Frontend teams để tối ưu timeline
- Đề xuất và triển khai event-driven architecture với EventBridge để tự động hóa email notifications và ranking updates

### 3. Tinh thần trách nhiệm
- Hoàn thành đúng 100% timeline đề ra: 5 tuần với 5 giai đoạn rõ ràng, không trễ deadline
- Đảm bảo code quality với 80%+ unit test coverage và integration testing đầy đủ
- Tự giác documentation đầy đủ với OpenAPI 3.0 specification và deployment guide

### 4. Hợp tác nhóm
- Làm việc hiệu quả trong mô hình parallel development với Backend/Frontend teams
- Tích cực tham gia daily standups (15 phút/ngày) và weekly sprint planning
- Hỗ trợ team members trong việc troubleshooting và code review

### 5. Đóng góp thực tế
- Xây dựng thành công platform với 100+ API operations và 85% code coverage
- Performance đạt chuẩn production: API response <500ms, chat latency <2s, system uptime ≥99%
- Chi phí tối ưu: $7-20/tháng với AWS Free Tier, phù hợp cho quy mô 50-100 sinh viên

## Điểm cần cải thiện

### 1. Tính kỷ luật
**Hiện trạng:** Đôi khi còn chủ quan trong việc tuân thủ một số quy trình nhỏ như commit message standards hoặc branch naming conventions.

**Kế hoạch cải thiện:**
- Thiết lập Git hooks để enforce commit message format và branch naming
- Sử dụng checklist trước mỗi pull request để đảm bảo tuân thủ đầy đủ coding standards
- Tham khảo và học hỏi từ các senior developers về discipline trong software development

### 2. Tư duy giải quyết vấn đề
**Hiện trạng:** Trong một số tình huống phức tạp (như Lambda cold start optimization hoặc DynamoDB query performance), tôi còn mất nhiều thời gian để tìm ra root cause do chưa có kinh nghiệm troubleshooting production issues.

**Kế hoạch cải thiện:**
- Học thêm về AWS troubleshooting techniques và best practices từ AWS documentation
- Thực hành debugging systematic với CloudWatch Logs, X-Ray tracing
- Xây dựng knowledge base cá nhân về common issues và solutions trong serverless architecture
- Tham gia AWS community forums để học hỏi từ real-world scenarios

### 3. Kỹ năng giao tiếp và trình bày
**Hiện trạng:** Khi trình bày technical concepts phức tạp (như DynamoDB GSI design patterns hoặc event-driven architecture), tôi đôi khi còn diễn đạt chưa rõ ràng và dễ hiểu cho người không có background kỹ thuật.

**Kế hoạch cải thiện:**
- Luyện tập cách simplify technical concepts bằng analogies và visual diagrams
- Thực hành presentation skills thông qua việc tạo technical blog posts hoặc video tutorials
- Học cách storytelling để làm cho technical presentations thú vị hơn
- Xin feedback sau mỗi lần trình bày để cải thiện liên tục

### 4. Khả năng quản lý thời gian
**Hiện trạng:** Trong giai đoạn đầu (Week 1-2), tôi còn underestimate effort cho một số tasks phức tạp như DynamoDB schema design với GSI, dẫn đến việc phải làm thêm giờ để catch up.

**Kế hoạch cải thiện:**
- Sử dụng time tracking tools (Toggl, Clockify) để phân tích thời gian thực tế cho từng loại task
- Áp dụng buffer time (thêm 20-30% estimated time) cho các tasks chưa quen
- Break down lớn tasks thành smaller chunks để estimate chính xác hơn
- Review và adjust estimates based on past performance data

## Kế hoạch phát triển bản thân

### Ngắn hạn (3-6 tháng)
- **Chứng chỉ:** Hoàn thành AWS Certified Developer Associate exam
- **Kỹ thuật:** Tìm hiểu sâu hơn về AWS advanced services (Step Functions, ECS/EKS, CloudFormation)
- **Soft skills:** Tham gia Toastmasters hoặc technical speaking clubs để cải thiện presentation skills
- **Thực hành:** Đóng góp vào open-source projects sử dụng AWS serverless để tích lũy kinh nghiệm

### Trung hạn (6-12 tháng)
- **Chứng chỉ:** Chuẩn bị và thi AWS Solutions Architect Professional
- **Chuyên môn:** Học về microservices architecture, container orchestration (Kubernetes)
- **Leadership:** Mentor các bạn junior trong team hoặc community projects
- **Networking:** Tham gia AWS User Groups, tech meetups để mở rộng network

### Dài hạn (1-2 năm)
- **Vai trò:** Phát triển theo hướng Cloud Solutions Architect hoặc DevOps Engineer
- **Chuyên sâu:** Trở thành expert trong một lĩnh vực cụ thể (serverless architecture, cost optimization, security)
- **Đóng góp:** Viết technical blogs, tạo video tutorials, hoặc speaking tại conferences
- **Impact:** Xây dựng và dẫn dắt các dự án có quy mô lớn hơn với team size 5-10 người

## Tổng kết

Thời gian thực tập tại AWS đã là một trải nghiệm vô cùng quý giá, giúp tôi không chỉ nâng cao kiến thức chuyên môn về cloud computing mà còn phát triển các kỹ năng mềm cần thiết cho một software engineer. Tôi đánh giá cao môi trường làm việc chuyên nghiệp, sự hỗ trợ tận tình từ mentor và team, cũng như cơ hội được làm việc với các công nghệ tiên tiến nhất.

Những gì tôi học được từ dự án Serverless Student Management Platform không chỉ là kỹ thuật, mà còn là cách tiếp cận professional trong software development: từ planning, design, implementation, testing, documentation, đến deployment và monitoring. Đây chính là nền tảng vững chắc để tôi tiếp tục phát triển sự nghiệp trong lĩnh vực Cloud Computing và Software Engineering.

Tôi cam kết sẽ tiếp tục học hỏi, cải thiện những điểm còn hạn chế, và phát huy những điểm mạnh để trở thành một Cloud Engineer/Solutions Architect xuất sắc trong tương lai.
